# Button
Button are used to initialize an action, either in the background or foreground of the app.
It comes in different Combinations and automaticly adjust to Dark or Light Mode
It can Display FeatherIcons and Loaders _That was an entire Week of my Life_

    <Collapse trigger="Variants" title="Everything from Primary to Soft" defaultExpanded>
        **The Button Can have one of these variants (declared as type="")**
        - type= "**primary**" - The Primary Button
        - type="**secondary**" - The Secondary Button
        - type="**tertiary**" - The Tertiary Button
        - type="**soft**" - The Soft Button
        - type="**text**" - The Text only Button Button

        - **outlined** - Changes Variant Pool to Outline Pool

    <Collapse trigger="Icons" title="Prefixes, Suffixes and Icons">
        **Prefixes, Suffixes and Icons use FeatherIcons as a Base**
        _This means that all FeatherIcons are valid Props_
        - **prefix**=<FeatherIcons.GitBranch size={12}/> - The Prefix will be displayed before the Button Text (Children)
        - children <FeatherIcons.GitCommit size={12} /> - The Button Text
        - **suffix**=<FeatherIcons.GitPullRequest size={12}/> - The Suffix will be displayed after the Button Text (Children)
        #### The Icon Prop
        **The Icon functions the same as the Prefix and Suffix but is build to be used as a Standalone Button with icon**
        it comes with auto collering and auto sizing

        <Button type='soft' outlined prefix='GitBranch'>Prefix</Button>
        <Button type='soft' outlined suffix='GitPullRequest'>Suffix</Button>
